# LP設計書

## 1. ページ構成
### 1.1 ヒーローセクション (Hero Section)
- **目的**: ユーザーの興味を一瞬で惹きつけ、利用開始へ誘導する。
- **要素**:
  - **キャッチコピー**: "AIがあなたの専属英語コーチに。瞬時に添削、確実に上達。" (例)
  - **サブコピー**: "Claude Haiku 4.5搭載。自然な英語表現と詳しい解説で、あなたのライティングを次のレベルへ。"
  - **CTAボタン**: "無料で始める" (ログイン/登録ページへ遷移)
  - **ビジュアル**: 英語学習やAIをイメージさせる抽象的なグラフィック、または利用シーンのイラスト。

### 1.2 サービス特徴 (Features)
- **目的**: ユーザーがこのサービスを使うメリットを伝える。
- **項目**:
  1.  **AIによる即時添削**: 待つことなく、その場ですぐにフィードバック。
  2.  **詳細な解説**: なぜ間違っているのか、どう直すべきかを丁寧に説明。
  3.  **学習履歴の管理**: 過去の添削内容を保存し、成長を可視化。

### 1.3 使い方ステップ (How it works)
- **目的**: 利用のハードルを下げる。
- **ステップ**:
  1.  **Step 1**: 英文を入力
  2.  **Step 2**: AIが瞬時にチェック
  3.  **Step 3**: 解説を読んで学習

### 1.4 フッター (Footer)
- **要素**:
  - 著作権表記
  - 利用規約、プライバシーポリシーへのリンク（プレースホルダー）
  - SNSリンク

## 2. デザイン要件
- **カラーパレット**:
  - メイン: 知的で信頼感のあるブルー系 (MUIのPrimaryカラー活用)
  - アクセント: 成長や活力を感じるオレンジやグリーン
  - 背景: ダークモード対応（白/黒、またはダークグレー）
- **タイポグラフィ**:
  - 読みやすさを重視したサンセリフ体 (Roboto等)
- **コンポーネント**:
  - Material UI (MUI) のコンポーネントを全面的に使用。
  - `Container`, `Box`, `Typography`, `Button`, `Grid`, `Card` 等。

## 3. 技術実装方針
- **ファイル**: `src/app/page.tsx`
- **レンダリング**: Server Component (必要に応じてClient Component部分を分離、あるいは全体を`use client`でMUI対応)
  - ※MUIのApp Router対応のため、今回はシンプルに`use client`を使用するか、スタイルが崩れないように注意する。
- **レスポンシブ**: MUIの`sx`プロパティや`Grid`システムを使用して、SP/PC両方で見やすく調整。
