# Google認証機能 動作確認チェックリスト

## 1. 画面表示
- [ ] **ログイン画面の表示**
  - `/auth/signin` にアクセスし、Googleログインボタンが表示されること。
  - デザインが崩れていないこと。
- [ ] **ヘッダーの表示（未ログイン時）**
  - ヘッダーに「Login」ボタンが表示されていること。
  - ユーザーアイコンやログアウトボタンが表示されていないこと。

## 2. ログイン動作
- [ ] **Googleログインの実行**
  - 「Sign in with Google」ボタンをクリックし、Googleの認証画面に遷移すること。
  - Googleアカウントでログイン後、ダッシュボード（`/dashboard`）にリダイレクトされること。
- [ ] **ヘッダーの表示（ログイン時）**
  - ヘッダーにユーザーアイコンとユーザー名が表示されること。
  - 「Logout」ボタンが表示されること。

## 3. データ保存
- [ ] **MongoDBへの保存**
  - 初回ログイン時に、MongoDBの `users` コレクションにユーザー情報（名前、Email、画像URL）が保存されていること。
  - 2回目以降のログインでは、新規データが作成されず、既存ユーザーとして扱われること。

## 4. ページ保護（ミドルウェア）
- [ ] **未ログイン時のアクセス制限**
  - ログアウト状態で `/dashboard` に直接アクセスした際、ログイン画面（`/auth/signin`）にリダイレクトされること。
- [ ] **ログイン時のアクセス許可**
  - ログイン状態で `/dashboard` にアクセスできること。

## 5. ログアウト動作
- [ ] **ログアウトの実行**
  - ヘッダーまたはダッシュボードの「Logout」ボタンをクリックし、ログアウトできること。
  - ログアウト後、トップページまたはログイン画面に遷移すること。
  - ログアウト後に `/dashboard` にアクセスすると、ログイン画面にリダイレクトされること。
