# 3. ユーザーニーズ洗い出し

## 機能的ニーズ（Functional Needs）
1. **英文添削機能**
   - 入力した英文の文法ミスを指摘してほしい。
   - より自然な表現（ネイティブらしい表現）を提案してほしい。
   - ビジネス、カジュアル、アカデミックなど、文脈に合わせた添削をしてほしい。

2. **解説機能**
   - なぜその修正が必要なのか、理由を説明してほしい。
   - 文法用語だけでなく、わかりやすい言葉で解説してほしい。

3. **履歴管理機能**
   - 過去に添削してもらった内容を後で見返したい。
   - 自分の弱点や傾向を知りたい。

4. **アカウント管理機能**
   - 自分のデータを安全に保存したい。
   - ログインしてどのデバイスからでもアクセスしたい。

5. **メール通知機能** (Nodemailer活用)
   - 学習のリマインダーや、定期的なレポートを受け取りたい（※MVPでは優先度低めかもしれないが、技術スタックにあるため考慮）。
   - パスワードリセットなどのシステム通知を受け取りたい。

## 非機能的ニーズ（Non-Functional Needs）
1. **パフォーマンス**
   - 添削結果が出るまでの待ち時間を短くしてほしい（Claude Haiku 4.5の高速性を活かす）。
   - サイトの動作が軽く、ストレスなく使いたい。

2. **ユーザビリティ**
   - 直感的に操作できるUIが良い（MUIの活用）。
   - スマホでもPCでも見やすく使いやすい（レスポンシブ対応）。

3. **信頼性・安全性**
   - 入力した英文（機密情報含む可能性）が漏洩しないようにしてほしい。
   - サービスが安定して稼働していてほしい。

4. **デザイン**
   - 学習意欲が湧くような、清潔感のあるモダンなデザインが良い。
