# 4. MVP機能一覧と優先順位

## 優先度定義
- **P0 (Must Have)**: MVPリリースに必須。これがないとサービスとして成立しない。
- **P1 (Should Have)**: あったほうが良い。P0が完了次第着手。
- **P2 (Nice to Have)**: 将来的に実装したい。今回はスコープ外の可能性が高い。

## 機能一覧

### 1. 認証・ユーザー管理 (NextAuth.js, MongoDB)
| ID | 機能名 | 優先度 | 概要 |
|---|---|---|---|
| A-01 | ユーザー登録 | P0 | メールアドレス/パスワード、またはGoogle認証での登録。 |
| A-02 | ログイン/ログアウト | P0 | セキュアなログイン状態の管理。 |
| A-03 | プロフィール設定 | P2 | ユーザー名、学習目標などの設定。 |

### 2. 英文添削 (Claude Haiku 4.5 API)
| ID | 機能名 | 優先度 | 概要 |
|---|---|---|---|
| C-01 | 英文入力フォーム | P0 | ユーザーが英文を入力するテキストエリア。文字数カウントなど。 |
| C-02 | AI添削実行 | P0 | Claude APIを呼び出し、添削結果を取得する。 |
| C-03 | 添削結果表示 | P0 | 修正前後の比較、修正理由の解説を表示する。 |
| C-04 | 文体指定オプション | P1 | ビジネス、カジュアル、アカデミックなどのトーンを選択可能にする。 |
| C-05 | 再添削（対話） | P2 | 添削結果に対して質問したり、別の表現を求めたりするチャット機能。 |

### 3. 履歴・データ管理 (MongoDB, Mongoose)
| ID | 機能名 | 優先度 | 概要 |
|---|---|---|---|
| H-01 | 添削履歴一覧 | P1 | 過去の添削内容をリスト表示する。 |
| H-02 | 履歴詳細閲覧 | P1 | 過去の添削結果の詳細を確認できる。 |
| H-03 | 履歴削除 | P2 | 不要な履歴を削除する。 |

### 4. その他・システム
| ID | 機能名 | 優先度 | 概要 |
|---|---|---|---|
| S-01 | メール通知 | P1 | 登録完了メールなどをNodemailerで送信する。 |
| S-02 | レスポンシブUI | P0 | PC/SP両対応のUI (MUI使用)。 |
| S-03 | LP (トップページ) | P0 | サービスの特徴を伝えるランディングページ。 |

## MVP開発スコープ (P0 + 一部のP1)
まずは「**ユーザーが登録・ログインし、英文を入力すると、AIが添削して結果を返す**」というコア体験を最優先で実装します。

1. **認証**: NextAuth.jsによるGoogle認証/メール認証
2. **コア機能**: 英文入力 -> Claude API連携 -> 結果表示
3. **UI**: MUIを使用したシンプルで使いやすいインターフェース
4. **DB**: MongoDBへのユーザー情報と添削ログの保存（履歴機能の基盤）
